# Working in this repo

An unattended GitHub Actions workflow (`.github/workflows/daily.yml`) commits
and pushes directly to `main` twice a day (roughly 08:00 and 21:45 UTC),
independent of whether James or Claude has this repo open. A separate
self-hosted runner (`Mac-snow-pred-accu`, James's own Mac — required because
Cloudflare blocks GitHub-hosted runner IPs for the Snowatch collector) also
pushes to `main` on the same schedule. So the local clone drifts out of sync
with `origin/main` routinely — this is expected, not a sign anything is
broken.

**Before making any edits in this repo, run `git fetch origin main` and check
whether local `main` is behind.** If it is, pull (fast-forward) before
starting work — don't wait to be asked. Doing this only once work is already
in progress risks a conflict in `docs/index.html`.

**`docs/index.html` is fully generated by `python dashboard.py`** — never
hand-resolve a diff or merge conflict in it. If it conflicts with an
incoming pull/merge, discard whichever side, complete the merge, then just
rerun `python dashboard.py` to regenerate it correctly from the merged
`data/snow.db` and current `dashboard.py`.
